# Majestic Boost

Majestic Boost — компактное Windows-приложение для подготовки системы к игре в Majestic / GTA V. Оно показывает понятное подтверждение перед первым применением настроек, запускает оптимизацию с правами администратора и просит перезагрузить компьютер, когда это действительно нужно.

## Что делает приложение

- применяет набор системных и игровых настроек из `MaxFPS-Apply.ps1`;
- сохраняет исходные значения перед изменением и поддерживает транзакционное восстановление при отмене или ошибке;
- не повторяет уже применённые изменения без необходимости;
- позволяет оставить запущенными Discord, Epic Games Launcher и Steam;
- запускает одноразовую игровую подготовку из `Game-Boost.ps1` и не закрывает заново программы, которые пользователь открыл после запуска Boost;
- содержит единый «Центр Boost» с проверкой готовности, отчётом последней сессии и настройками;
- по выбору сохраняет OneDrive, Teams, Wallpaper Engine и NVIDIA Overlay;
- пока Boost активен, обнаруживает GTA V / GTA V Enhanced и временно повышает только её приоритет до безопасного `AboveNormal`;
- раз в две минуты проверяет давление памяти и при необходимости освобождает только неиспользуемую управляемую память самой Majestic Boost, не вмешиваясь в память игры и системный standby-кэш;
- при остановке возвращает исходный приоритет, но не перезаписывает изменение, которое сделал пользователь или другая программа;
- поддерживает выключенную по умолчанию автоматическую сессию при обнаружении GTA, пока открыто приложение;
- сохраняет локальный отчёт о выполненных, пропущенных и восстановленных действиях;
- выполняет ручной 60-секундный тест FPS через официальный PresentMon 2.5.1 и показывает средний FPS, 1% low, P95/P99 и статтеры;
- позволяет из Центра Boost вручную открыть безопасное восстановление системных настроек;
- позволяет вторым нажатием остановить активный контроль и вернуть ракету в неактивное состояние без отката постоянных системных настроек;
- при каждом запуске проверяет подписанный манифест обновления;
- показывает реальный процент скачивания и установки обновления;
- устанавливает обязательное обновление только после проверки RSA-подписи манифеста, версии, размера и SHA-256 установщика;
- после успешного обновления предлагает одной кнопкой продолжить работу в новой версии.

Приложение не является частью Majestic и не подменяет файлы игры. Оно не внедряется в процесс GTA и не очищает принудительно working set/standby list. Некоторые операции меняют системные параметры Windows и требуют подтверждения UAC.

## Сборка

Требуется Windows с .NET Framework 4.8 и 64-битным Framework C# compiler.

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\build.ps1
```

Результат появится в `dist`:

- `MajesticBoost.exe` — бинарник приложения для тестирования и установщика;
- `MajesticBoost-Setup-1.6.3.exe` — рекомендуемый установщик;
- `MajesticBoost-Setup-Latest.exe` — стабильное имя актуального универсального установщика.

Установщик также публикует закреплённый официальный `PresentMon.exe` вместе с `LICENSE.txt` и `THIRD_PARTY.txt` в `Tools\PresentMon`. Перед сборкой и запуском его размер и SHA-256 проверяются.

## Выпуск обновления

1. Соберите и полностью проверьте новую версию.
2. Обновите версию в приложении, установщике и manifest-файлах.
3. Рассчитайте SHA-256 и размер нового установщика и обновите `update-v2.json`.
4. Подпишите точные байты `update-v2.json` скриптом `tools/Sign-UpdateManifest.ps1`.
5. Опубликуйте установщик, `MajesticBoost-Setup-Latest.exe`, `update-v2.json` и `update-v2.json.sig` одним проверенным изменением.

`update.json` и `update.json.sig` являются замороженным legacy-каналом для версий со старым дефектным обновлятором. Их нельзя снова направлять на обязательный релиз: такие клиенты не способны пройти собственную проверку установщика. Для одноразового перехода используется `MajesticBoost-Setup-Latest.exe`; начиная с 1.5.1 приложение работает через исправленный канал `update-v2.json`.

PresentMon распространяется по MIT. Его исходный проект и уведомления: <https://github.com/GameTechDev/PresentMon>.

Приватный ключ подписи хранится только локально в зашифрованном DPAPI-файле и никогда не должен добавляться в репозиторий. Приложение содержит только публичный ключ.
